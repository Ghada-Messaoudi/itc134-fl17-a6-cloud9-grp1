Navigation in UNIX

Essential navigation commands in Unix

  • * pwd – confirm the current directory you're in

  • * ls – list files and directories

  • * cd – change directory Additional commands

While not helping in your navigation, these commands will definitely help you learn more about where and what you're looking at in your file system tree:

  • * file – confirm the type of a file

  • * find – find files and directories

  • * locate – quickly find files and directories using locatedb database

  • * dirname – extract the directory name from a full path to a file

  • * basename – strip directory and suffix from a full path to a file

No matter if you are a developer or a normal UNIX user, creating shortcuts for your usual tasks is a very helpful thing that will gain you precious time when navigating and typing. Using alias is the key to achieving speed and efficiency in UNIX. And if you want to get rid of an alias, just use the unalias commandThese some basic alias that helps a lot in navigation:

  • * alias up='cd .. ; pwd'

  • * alias up2='cd ../.. ; pwd'

  • * alias up3='cd ../../.. ; pwd'

  • * alias chrome="open -a \"Google Chrome\""

  • * alias folders="find . -maxdepth 1 -type d -print | xargs du -sk | sort -rn" (Shows the size (sorted) of only the folders in this directory)

pushd and popd: pushd adds a directory to the top of the stack and popd removes a directory from the top of the stack.The pushd command saves the current working directory in memory so it can be returned to at any time, optionally changing to a new directory. The popd command returns to the path at the top of the directory stack. This directory stack is accessed by the command dirs in Unix

  • * Pushd command – puts/adds directory paths onto a directory stack (history) and later allowing you to navigate back to any directory in history. While you add directories to the stack, it also echoes what’s existing in history (or “stack”).

  • * Popd command – removes a directory from the top of the stack or history.
